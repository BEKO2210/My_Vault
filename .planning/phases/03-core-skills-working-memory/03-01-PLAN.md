Create `.agents/skills/create/create-utils.cjs` with these exported functions:

**`getTemplateInfo(type)`** -- Returns `{ template, folder, prefix }` for a given note type. Use the TEMPLATE_MAP object from the research (12 entries matching the CLAUDE.md table exactly). Returns `null` for unknown types.

**`buildFileName(type, title, date)`** -- Constructs the correct filename per naming conventions:
- `daily` type: `{date}.md` (e.g., `2026-03-07.md`)
- `meeting` type: `Meeting -- {title} {date}.md`
- `weekly` type: `Weekly Review {date}.md`
- `monthly` type: `Monthly Review {date}.md`
- All others: `{prefix}{title}.md` where prefix comes from TEMPLATE_MAP (e.g., `Tool -- ` for tools)

**`getDateVars(dateStr)`** -- Takes a `YYYY-MM-DD` string, returns `{ date, yesterday, tomorrow, time }`. Use `new Date(dateStr + 'T12:00:00')` to avoid timezone issues. `time` = current time as `HH:mm`. `yesterday` and `tomorrow` computed via `setDate()` and formatted as `YYYY-MM-DD`.

**`substituteVariables(templateContent, vars)`** -- Replaces all template variables:
- `{{date}}` with `vars.date`
- `{{title}}` with `vars.title`
- `{{time}}` with `vars.time`
- `{{yesterday}}` with `vars.yesterday`
- `{{tomorrow}}` with `vars.tomorrow`
- `{{date:YYYY-MM-DD}}` with `vars.date`
- `{{date:MMMM YYYY}}` with full month name + year (e.g., "March 2026")
- `{{date:ww}}` with ISO week number
All replacements are global (`/g` flag). The function handles the Obsidian Templater-style `{{date:FORMAT}}` variants found in Monthly Review and Weekly Review templates.

Helper: **`getISOWeekNumber(date)`** -- Computes ISO 8601 week number. Standard algorithm: find nearest Thursday, compute week count from Jan 1.

Helper: **`getMonthName(monthIndex)`** -- Returns English month name for 0-11 index.

**`suggestWikiLinks(noteType, noteTags, vaultIndex, tagIndex, limit)`** -- Suggests 3-5 wiki-links for a newly created note:
1. Score by shared tags: for each tag in `noteTags`, look up notes in `tagIndex.tags[tag]`, score +1 per shared tag
2. Filter out templates (`isTemplate: true`), MOCs, and system files
3. Boost the relevant MOC for the note type (+2 score): map noteType to MOC name (tool->"Tools MOC", project->"Projects MOC", etc.)
4. Sort by score descending, return top `limit` (default 5) results as `[{ path, name, score, reasons }]`

**`mocUsesDataview(vaultRoot, mocPath)`** -- Reads a MOC file and returns `true` if it contains `` ```dataview `` blocks. Used to determine whether to manually add links to a MOC (only for static MOCs; Dataview MOCs auto-list via queries).

**`ensureFreshIndexes(vaultRoot)`** -- Checks scan-state.json timestamp. If indexes are older than 5 minutes (STALE_MS = 300000), calls `scan()` from scanner.cjs. Returns the scan result or null if fresh.

Use `require` for all imports (CommonJS .cjs). Import `fs`, `path` from Node.js built-ins. Import `loadJson` from `../scan/utils.cjs`. Import `scan` from `../scan/scanner.cjs`.

`module.exports = { getTemplateInfo, buildFileName, getDateVars, substituteVariables, suggestWikiLinks, mocUsesDataview, ensureFreshIndexes };`

  <action>
Create `.agents/skills/daily/daily-utils.cjs` with these exported functions:

**`extractOpenItems(vaultRoot, today)`** -- Scans daily notes from the past 7 days for unchecked checkboxes:
1. Loop from 1 to 7 days back from `today` (YYYY-MM-DD string)
2. For each day, check if `{vaultRoot}/00 - Inbox/Daily Notes/{dateStr}.md` exists
3. Read file content, split into lines
4. Match lines with regex `^[-*]\s+\[ \]\s+(.+)$` (unchecked checkboxes)
5. Skip items that already contain `(from [[` -- these are previously rolled-over items
6. Return array of `{ text, source, formatted }` where `formatted` = `- [ ] {text} (from [[{dateStr}]])`
Per user decision: only unchecked checkboxes (`- [ ]`) count as open items.

**`formatRolloverSection(items)`** -- Takes the array from extractOpenItems and formats a `## Rolled Over` section:
```
## Rolled Over

- [ ] Task text (from [[2026-03-06]])
- [ ] Another task (from [[2026-03-04]])
```
Returns empty string if no items.

**`dailyNoteExists(vaultRoot, dateStr)`** -- Returns `true` if `{vaultRoot}/00 - Inbox/Daily Notes/{dateStr}.md` exists.

**`mergeRolloverItems(existingContent, newItems)`** -- For the case where today's daily note already exists (user decision: merge non-destructively):
1. Parse existing content for a `## Rolled Over` section
2. Extract existing rollover task texts (strip the `(from [[...]])` suffix for comparison)
3. Filter `newItems` to only those whose `text` is not already present
4. If `## Rolled Over` section exists, append new items to it
5. If no `## Rolled Over` section exists, insert one before `## Connections` (or at end)
6. Return the updated full content

Also re-export from create-utils.cjs: **`getDateVars`** and **`substituteVariables`** -- so the /daily SKILL.md can reference a single module. Actually, import them from `../create/create-utils.cjs` and re-export. This avoids code duplication.

Use `require` for all imports. Import `fs`, `path` from Node.js built-ins.

`module.exports = { extractOpenItems, formatRolloverSection, dailyNoteExists, mergeRolloverItems, getDateVars, substituteVariables };`
