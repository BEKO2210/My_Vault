Create `.agents/skills/connect/connect-utils.cjs` with these exported functions:

**`findConnections(targetPath, vaultIndex, linkMap, tagIndex)`** -- Discovers related notes for a given note:

1. Look up the target note in `vaultIndex.notes[targetPath]`
2. Get the target's tags from `allTags`
3. Get the target's existing outbound links from `links`
4. Build a score map: `{ notePath: { score, evidence[] } }`
5. **Tag overlap scoring:** For each tag in target's allTags, look up `tagIndex.tags[tag]`. For each note with that tag (excluding targetPath, templates, MOCs, system files), add +1 to score and push `'shared tag: #' + tag` to evidence.
6. **Link adjacency scoring:** For each link in linkMap.links, if another note links to the same target as the target note, add +0.5 and push `'both link to [[' + sharedTarget + ']]'` to evidence.
7. **Exclude already-linked notes:** Filter out notes that the target already directly links to (avoid suggesting existing connections).
8. Sort by score descending.
9. Return top results as `[{ path, name, score, evidence, confidence }]` where confidence is:
   - score >= 3: "high"
   - score >= 1.5: "medium"
   - score > 0: "low"

**`formatConnectionSuggestions(suggestions)`** -- Formats the suggestions for display:
```
1. **[[Note Name]]** (high confidence)
   - shared tag: #tool
   - shared tag: #productivity
   - both link to [[Home]]

2. **[[Another Note]]** (medium confidence)
   - shared tag: #project
```
Returns a formatted string.

**`ensureFreshIndexes(vaultRoot)`** -- Same pattern as in create-utils: check scan-state.json timestamp, call scan() if stale (>5 min). Import from `../scan/scanner.cjs` and `../scan/utils.cjs`.

Use `require` for all imports. Import `fs`, `path`. Import `loadJson` from `../scan/utils.cjs`. Import `scan` from `../scan/scanner.cjs`.

<task type="auto">
  <name>Task 2: Create health-utils.cjs utility module</name>

  <action>
Create `.agents/skills/health/health-utils.cjs` with these exported functions:

**`analyzeHealth(vaultIndex, linkMap)`** -- Performs full vault health analysis:

*Orphan detection:*
1. Build a connection count map from linkMap.links: for each resolved link, count +1 for source and +1 for targetPath
2. For each note in vaultIndex.notes: if NOT a template, NOT a MOC/home/system type, and connection count <= 1, mark as orphan
3. Return orphans as `[{ path, name, connections }]`

*Broken link detection:*
1. Filter linkMap.links for entries where `resolved === false`
2. Exclude links from template files (isTemplate: true in source note)
3. For each broken link, generate fix suggestions via `suggestFixes(target, knownNames)`
4. Return broken links as `[{ source, target, suggestions }]`

Return `{ orphans, brokenLinks, stats }` where stats = `{ totalNotes, totalLinks, orphanCount, brokenLinkCount }`.

**`suggestFixes(brokenTarget, knownNames)`** -- Generates fix suggestions for a broken link target:
1. Case-insensitive substring match: any known name that contains the target or vice versa
2. Levenshtein distance <= 2: catches typos (e.g., "Docekr" -> "Docker")
3. Return up to 3 suggestions sorted by relevance (exact substring first, then Levenshtein)

**`levenshtein(a, b)`** -- Standard Levenshtein distance implementation (~15 lines). Dynamic programming matrix approach. Operates on lowercased strings for case-insensitive comparison.

**`ensureFreshIndexes(vaultRoot)`** -- Same pattern as other skills.

**`classifyFix(suggestions)`** -- Determines whether a broken link fix can be auto-applied:
- If exactly 1 suggestion with Levenshtein distance <= 1: return `{ action: 'auto', fix: suggestion }` (AUTO zone -- obvious typo)
- If 1+ suggestions but ambiguous: return `{ action: 'propose', candidates: suggestions }` (PROPOSE zone)
- If 0 suggestions: return `{ action: 'manual', reason: 'no match found' }` (user decides)
Per user decision: auto-fix when single clear match, propose when ambiguous.

Use `require` for all imports. Import `path`. Import `loadJson` from `../scan/utils.cjs`. Import `scan` from `../scan/scanner.cjs`.

`module.exports = { analyzeHealth, suggestFixes, levenshtein, classifyFix, ensureFreshIndexes };`

**Create `.agents/skills/health/SKILL.md`:**

Frontmatter: `name: health`, `trigger: /health`, `description: Detect orphan notes and broken wiki-links with suggested fixes`, `version: 1.0.0`

Sections:
1. **Header**: `# /health -- Vault Health Check`
2. **Description**: Quick diagnostic of vault health. Reports orphan notes (0-1 connections) and broken wiki-links with suggested fixes. Auto-fixes obvious typos, proposes fixes for ambiguous cases.
3. **Usage**:
   - `/health` -- Full vault health report
   - `/health --fix` -- Auto-apply all auto-fixable broken links (single clear match, Levenshtein <=1)
4. **Execution Flow** (numbered steps Claude follows):
   1. Ensure fresh indexes: call `ensureFreshIndexes('.')`
   2. Load vault-index.json and link-map.json
   3. Call `analyzeHealth(vaultIndex, linkMap)`
   4. Display summary stats at top: "X orphan notes, Y broken links, Z total notes"
   5. **Orphan section:** List each orphan with its current connection count and suggest potential links (run a mini findConnections for each orphan to suggest links based on its tags)
   6. **Broken links section:** For each broken link:
      a. Call `classifyFix(suggestions)` to determine action
      b. If `auto`: show the fix and mark it for auto-application ("Will fix: [[Docekr]] -> [[Docker]]")
      c. If `propose`: show candidates and ask user ("Did you mean: [[Docker]] or [[Docker Compose]]?")
      d. If `manual`: flag for user attention ("No match found for [[Nonexistent]]")
   7. If `--fix` flag: apply all auto-classified fixes by reading source files, replacing broken link text, writing back
   8. Call `scan('.')` to update indexes after fixes
   Per user decision: summary first, then actionable list. Auto-fix obvious (AUTO zone), propose ambiguous (PROPOSE zone).
5. **Report Format**: Show the summary stats at top, then orphans section, then broken links section.
6. **Governance**: Auto-fix follows AUTO zone (fix broken links when obvious). Propose follows PROPOSE zone (ask when ambiguous). Log auto-fixes to `.claude/changelog.md`.
7. **Code Example**: JavaScript snippet using health-utils.cjs.
8. **Limitations**: Does not detect semantic orphans (notes with links but no meaningful content overlap). Does not check frontmatter consistency (deferred to /maintain in Phase 5).
