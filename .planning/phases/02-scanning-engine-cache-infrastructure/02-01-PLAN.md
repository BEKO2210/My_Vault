Create the `.agents/skills/scan/` directory structure.

**utils.cjs** -- Shared utilities with these exports:

1. `hashContent(content)` -- SHA-256 hash using Node.js built-in `crypto.createHash('sha256').update(content, 'utf8').digest('hex')`.

2. `writeJsonAtomic(filePath, data)` -- Write to `filePath + '.tmp'` first, then `fs.renameSync(tmpPath, filePath)`. Create parent directory with `{ recursive: true }` if it doesn't exist.

3. `discoverFiles(vaultRoot)` -- Use `fs.globSync('**/*.md', { cwd: vaultRoot })`. Normalize all paths to forward slashes with `.replace(/\\/g, '/')`. `fs.globSync` excludes dot-directories (`.*`) by default, so `.claude/`, `.obsidian/`, `.planning/`, `.agents/` are automatically excluded. Return sorted array of vault-relative paths.

4. `normalizePath(p)` -- Replace backslashes with forward slashes: `p.replace(/\\/g, '/')`.

5. `loadJson(filePath)` -- Read and parse JSON file, return null if file doesn't exist or is invalid.

Use only Node.js built-ins: `fs`, `path`, `crypto`. Zero external dependencies. Use `.cjs` format with `module.exports`.

**parser.cjs** -- Level 1 metadata extraction with these exports:

1. `extractFrontmatter(content)` -- Hand-rolled YAML parser for flat key-value + array format. Handles:
   - Simple key-value: `type: project` -> `{ type: "project" }`
   - Array (YAML list): `tags:\n  - foo\n  - bar` -> `{ tags: ["foo", "bar"] }`
   - Inline array: `tags: [foo, bar]` -> `{ tags: ["foo", "bar"] }`
   - Empty values: `tags:` or `tags: []` -> `{ tags: [] }`
   - Strip quotes from values: `type: "project"` -> `type: "project"`
   - Strip leading `#` from tags: `- #project` -> `"project"`
   - Return `{ type, tags, frontmatter }` where `type` is the `type` field (or null), `tags` is the tags array (or []), and `frontmatter` is the full parsed object.
   - If no frontmatter found (no `---` delimiters), return `{ type: null, tags: [], frontmatter: {} }`.
   - Wrap in try-catch: on malformed YAML, return the default empty result (do not crash).
   - Use regex `^---\r?\n([\s\S]*?)\r?\n---` to extract the YAML block.

2. `extractWikiLinks(content)` -- Extract all wiki-links with this process:
   - First, strip fenced code blocks: `content.replace(/```[\s\S]*?```/g, '')`
   - Then strip inline code: `.replace(/`[^`]+`/g, '')`
   - Apply regex `(?<!!)\[\[([^\]]+?)\]\]` with global flag (the `(?<!!)` negative lookbehind excludes embeds `![[...]]`)
   - For each match, split on `|` for alias, split target on `#` for heading reference
   - Handle Obsidian escaped pipe `\|` inside Dataview table wiki-links
   - Return array of `{ target, alias, heading }` objects (alias and heading are null when absent)
   - Skip empty `[[]]` matches

3. `extractHeadings(content)` -- Extract headings from body (after frontmatter):
   - Remove frontmatter block first: `content.replace(/^---\r?\n[\s\S]*?\r?\n---\r?\n?/, '')`
   - Strip fenced code blocks
   - Apply regex `^(#{1,6})\s+(.+)$` with global and multiline flags
   - Return array of `{ level, text }` objects

4. `extractInlineTags(content)` -- Extract inline `#tags` from body text:
   - Remove frontmatter block first
   - Strip fenced code blocks and inline code
   - Apply regex `(?:^|[\s>])#([\w][\w/-]*)` with global flag
   - Return deduplicated array of tag strings (without the `#` prefix)
   - Exclude common false positives: headings (lines starting with `#` followed by space are headings, not tags)

5. `parseFile(vaultRoot, relativePath)` -- Orchestrate all extractors for a single file:
   - Read file content with `fs.readFileSync(fullPath, 'utf8')`
   - Get stat with `fs.statSync(fullPath)`
   - Call all extractors
   - Determine `isTemplate` flag: `true` if path starts with `05 - Templates/`
   - Merge frontmatter tags and inline tags into a combined `allTags` array (deduplicated)
   - Return object: `{ path, name, type, tags (frontmatter only), inlineTags, allTags, links, headings, frontmatter, hash, mtime, scanned, isTemplate }`
   - `path` uses forward slashes (vault-relative)
   - `name` is `path.basename(relativePath, '.md')`
   - `hash` from `hashContent(content)`
   - `mtime` from `stat.mtimeMs`
   - `scanned` is `Date.now()`

Use the code examples from 02-RESEARCH.md as reference but adapt as needed. All functions must be robust against malformed input (try-catch where appropriate, never crash on a single bad file).

Create `.agents/skills/scan/scanner.cjs` that orchestrates the full scan pipeline.

**scanner.cjs** exports a single function `scan(vaultRoot, options)`:

`options` object:
- `full` (boolean, default false) -- Force full re-scan ignoring hashes
- `verbose` (boolean, default false) -- Return per-file details in results
- `indexDir` (string, default `path.join(vaultRoot, '.claude', 'indexes')`) -- Where to write index files

**scan() implementation -- follow the Two-Phase Scan pattern from RESEARCH.md:**

1. **Discover:** Call `discoverFiles(vaultRoot)` to get all current .md files.

2. **Load previous state:** Call `loadJson(path.join(indexDir, 'scan-state.json'))` and `loadJson(path.join(indexDir, 'vault-index.json'))`.

3. **Classify changes:** Implement `classifyChanges(vaultRoot, currentFiles, prevState, forceFull)`:
   - If `forceFull` or no previous state: all files are "added"
   - Otherwise, for each current file:
     - Not in prevState.files -> "added"
     - In prevState.files but mtime differs -> check hash: if hash differs -> "modified", else -> "unchanged"
     - In prevState.files and mtime matches -> "unchanged" (fast path)
   - Files in prevState.files but not in currentFiles -> "deleted"
   - Return `{ added, modified, deleted, unchanged }` (arrays of relative paths)

4. **Parse changed files:** For each file in `[...added, ...modified]`, call `parseFile(vaultRoot, file)`.

5. **Merge results:** Build the final notes object:
   - For parsed files: use new parse results
   - For unchanged files: carry forward entries from previous vault-index.json
   - For deleted files: exclude them
   - Store as object keyed by normalized path: `{ [path]: noteMetadata }`

6. **Write vault-index.json:** Using `writeJsonAtomic`:
   ```json
   {
     "version": 1,
     "generated": Date.now(),
     "noteCount": Object.keys(notes).length,
     "notes": { ... }
   }
   ```

7. **Write scan-state.json:** Using `writeJsonAtomic`:
   ```json
   {
     "version": 1,
     "lastScan": Date.now(),
     "lastFullScan": (if full scan) Date.now() (else carry forward),
     "scanCount": prevState ? prevState.scanCount + 1 : 1,
     "files": {
       [path]: { "hash": "...", "mtime": ... }
     }
   }
   ```
   Populate `files` from the merged notes (hash and mtime for every note in the final index).

8. **Return results object:**
   ```javascript
   {
     added: added.length,
     modified: modified.length,
     deleted: deleted.length,
     unchanged: unchanged.length,
     total: Object.keys(notes).length,
     elapsed: Date.now() - startTime,
     // if verbose: include arrays of file paths
     details: options.verbose ? { added, modified, deleted, unchanged } : undefined
   }
   ```

**Important implementation details:**
- Require parser and utils from same directory: `require('./parser.cjs')`, `require('./utils.cjs')`
- Create `.claude/indexes/` directory if it doesn't exist (writeJsonAtomic handles this)
- Do NOT write link-map.json or tag-index.json here -- that's Plan 02
- All paths in JSON files use forward slashes
- The scanner is READ-ONLY for vault files -- it never modifies .md files
- Wrap the entire scan in try-catch to surface errors cleanly
