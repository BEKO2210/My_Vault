Build the embedding infrastructure and /search skill -- enabling semantic search that finds vault notes by meaning, not just keywords.

  <action>
Create `.agents/skills/search/embedder.cjs` with these exported functions:

**SQLite Operations:**

**`openDb(vaultRoot)`** -- Opens (or creates) the SQLite database at `{vaultRoot}/.claude/embeddings.db`. Uses `require('node:sqlite').DatabaseSync`. Creates parent directory if needed. Runs schema creation:
```sql
CREATE TABLE IF NOT EXISTS embeddings (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  path TEXT UNIQUE NOT NULL,
  embedding BLOB NOT NULL,
  content_hash TEXT NOT NULL,
  note_type TEXT,
  title TEXT,
  updated INTEGER NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_emb_path ON embeddings(path);
CREATE INDEX IF NOT EXISTS idx_emb_hash ON embeddings(content_hash);
```
Sets `PRAGMA journal_mode=WAL` for better concurrent read performance. Returns the db instance.

**`storeEmbedding(db, notePath, embedding, contentHash, noteType, title)`** -- Stores a Float32Array embedding as BLOB. Convert via `new Uint8Array(embedding.buffer)`. Uses `INSERT OR REPLACE` to upsert. Sets `updated` to `Date.now()`.

**`getAllEmbeddings(db)`** -- Returns all rows: `SELECT path, embedding, title, note_type FROM embeddings`.

**`deleteEmbedding(db, notePath)`** -- Deletes embedding for a given path.

**`blobToVector(blob)`** -- Converts a retrieved BLOB (Uint8Array) back to Float32Array: `new Float32Array(blob.buffer, blob.byteOffset, blob.byteLength / 4)`.

**`getEmbeddingStatus(vaultRoot)`** -- Opens db, queries count, latest update time, oldest update time. Returns `{ totalEmbeddings, latestUpdate, oldestUpdate }`. Closes db. Returns `{ totalEmbeddings: 0, latestUpdate: null, oldestUpdate: null }` if db doesn't exist or is empty.

**Embedding Generation:**

**`generateEmbedding(text)`** -- Async function. Uses dynamic `import('@huggingface/transformers')` to load the ESM package from CJS. Caches the pipeline in a module-level `_pipeline` variable. Creates pipeline with `pipeline('feature-extraction', 'Xenova/all-MiniLM-L6-v2', { cache_dir: path.join(vaultRoot || '.', '.claude', '.models') })` on first call. Generates embedding with `{ pooling: 'mean', normalize: true }`. Returns `new Float32Array(output.data)`.

Important: The dynamic `import()` MUST be wrapped in try/catch. If it fails (package not installed), throw a descriptive error: `'@huggingface/transformers not installed. Run: npm install @huggingface/transformers'`.

**`isEmbeddingAvailable()`** -- Async function. Tries `await import('@huggingface/transformers')`. Returns `true` if successful, `false` if not. Does NOT throw. Caches the result in a module-level boolean.

**Text Extraction:**

**`extractBodyText(content)`** -- Strips markdown to plain text for embedding input:
1. Remove YAML frontmatter: `/^---\r?\n[\s\S]*?\r?\n---\r?\n?/`
2. Remove fenced code blocks: `/```[\s\S]*?```/g`
3. Remove inline code: `/`[^`]+`/g`
4. Convert wiki-links to plain text: `[[Note|Display]]` -> `Display`, `[[Note]]` -> `Note`
5. Remove markdown formatting (bold, italic): `/[*_~]{1,3}/g`
6. Remove heading markers: `/^#{1,6}\s+/gm`
7. Remove HTML tags: `/<[^>]+>/g`
8. Collapse excess whitespace: `/\n{3,}/g` -> `\n\n`
9. Trim

**`buildEmbeddingInput(noteName, noteContent, maxLength)`** -- Combines note name and body text for embedding. `maxLength` defaults to 512 (safe range for MiniLM's 256-token limit). Returns `(noteName + '. ' + extractBodyText(noteContent)).slice(0, maxLength)`.

**Incremental Sync:**

**`syncEmbeddings(vaultRoot, scanResult, vaultIndex)`** -- Async function. Called after /scan to update embeddings for changed files.
1. Check `await isEmbeddingAvailable()`. If false, return `{ skipped: true, reason: 'transformers not installed' }`.
2. Open db via `openDb(vaultRoot)`.
3. Determine files to embed: `scanResult.details?.added || []` + `scanResult.details?.modified || []`. If `scanResult.details` is undefined (non-verbose scan), fall back to embedding all files in vaultIndex that don't have a current embedding (compare content_hash).
4. Determine files to remove: `scanResult.details?.deleted || []`.
5. Delete embeddings for removed files.
6. For each file to embed:
   - Skip if `note.isTemplate` is true
   - Skip if path starts with `'05 - Templates/'` or `'06 - Atlas/'`
   - Skip system files: `Home.md`, `START HERE.md`, `Workflow Guide.md`, `Tag Conventions.md`
   - Read file content via `fs.readFileSync(path.join(vaultRoot, filePath), 'utf8')`
   - Build embedding input via `buildEmbeddingInput(note.name, content)`
   - Generate embedding via `await generateEmbedding(text)`
   - Store via `storeEmbedding(db, filePath, embedding, note.hash, note.type, note.name)`
7. Close db.
8. Return `{ embedded: count, deleted: count, total: db total }`.

Use `require` for all imports except `@huggingface/transformers` (dynamic `import()`). Import `fs`, `path` from Node.js built-ins. Import `{ DatabaseSync }` from `node:sqlite`.

`module.exports = { openDb, storeEmbedding, getAllEmbeddings, deleteEmbedding, blobToVector, generateEmbedding, isEmbeddingAvailable, extractBodyText, buildEmbeddingInput, syncEmbeddings, getEmbeddingStatus };`

Create `.agents/skills/search/search-utils.cjs` with these exported functions:

**`cosineSimilarity(a, b)`** -- Computes cosine similarity between two Float32Arrays. Standard dot-product / (norm_a * norm_b) formula. Returns 0 if either vector has zero norm. Tight loop for performance (no array methods -- use for-loop with index access).

**`semanticSearch(queryEmbedding, allEmbeddings, options)`** -- Searches all stored embeddings for semantic similarity to a query vector.
- `options.threshold` defaults to 0.3 (minimum similarity to include -- per user decision: adaptive result count based on relevance threshold)
- `options.maxResults` defaults to 20
- For each row in `allEmbeddings`:
  1. Convert blob to Float32Array via `blobToVector(row.embedding)` (import from embedder.cjs)
  2. Compute `cosineSimilarity(queryEmbedding, stored)`
  3. If similarity >= threshold, add to results
- Each result: `{ path, title, type: row.note_type, similarity, confidence }` where confidence is:
  - similarity >= 0.7: `'high'`
  - similarity >= 0.5: `'medium'`
  - similarity > 0: `'low'`
- Sort by similarity descending, slice to maxResults.

**`keywordSearch(query, vaultIndex, tagIndex)`** -- Fallback search when embeddings are unavailable. Searches by:
1. **Tag matching:** Tokenize query into words (lowercase, split on spaces). For each word, check if it exists as a tag in `tagIndex.tags`. Collect matching note paths with score +2 per tag match.
2. **Title matching:** For each note in `vaultIndex.notes`, check if any query word appears in `note.name.toLowerCase()`. Score +1 per word match.
3. Filter out templates, MOCs (path starts with `06 - Atlas/`), and system files.
4. Sort by score descending. Return top 10 results as `[{ path, title, type, score, confidence, method: 'keyword' }]`.
- Confidence: score >= 4 -> high, score >= 2 -> medium, else low.

**`extractExcerpt(bodyText, query, maxLength)`** -- Extracts a relevant excerpt from note body text that shows why it matched.
- `maxLength` defaults to 150 characters
- Algorithm:
  1. Tokenize query into words (lowercase, split on spaces, filter words < 3 chars)
  2. Search bodyText for the first occurrence of any query word (case-insensitive)
  3. If found: extract a window of `maxLength` chars centered on that occurrence, add `...` prefix/suffix if truncated
  4. If no query word found: return first `maxLength` chars of bodyText with `...` suffix if truncated
- Returns the excerpt string (always non-empty if bodyText is non-empty)

**`formatSearchResults(results, query, vaultRoot)`** -- Formats search results for display to the user. Per user decision: note title + relevant excerpt showing why it matched + relevance indicator (high/medium/low).
- For each result, if `result.excerpt` is not already set, reads the note file via `fs.readFileSync(path.join(vaultRoot, result.path), 'utf8')`, extracts body text via `extractBodyText` (imported from embedder.cjs), and generates excerpt via `extractExcerpt(bodyText, query)`. Wraps file read in try/catch (excerpt becomes empty string on failure).
- Import `extractBodyText` from `./embedder.cjs` at the top of the file.

Format:
```
## Search: "{query}"

Found {N} results:

1. **[[Note Title]]** (high confidence)
   > ...relevant excerpt showing why this note matched...
   Type: tool | Relevance: 0.82

2. **[[Another Note]]** (medium confidence)
   > ...another relevant excerpt...
   Type: resource | Relevance: 0.56

...

_Searched {total} notes via {method}_
```
Where `method` is "semantic similarity" or "keyword matching" depending on whether results have `similarity` or `score` fields.

**`ensureFreshIndexes(vaultRoot)`** -- Same pattern as other skills: check scan-state.json timestamp, call scan() if stale (>5 min). Import from `../scan/scanner.cjs` and `../scan/utils.cjs`.

Use `require` for all imports. Import `fs`, `path` from Node.js built-ins. Import `{ blobToVector, extractBodyText }` from `./embedder.cjs`. Import `{ loadJson }` from `../scan/utils.cjs`. Import `{ scan }` from `../scan/scanner.cjs`.
