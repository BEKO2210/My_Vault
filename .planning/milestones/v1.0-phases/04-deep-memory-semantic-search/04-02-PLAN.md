Create `.agents/skills/memory/memory-utils.cjs` with these exported functions:

**Insight Distillation:**

**`parseInsights(vaultRoot)`** -- Reads `.claude/memory/insights.md` and parses it into a structured object.
- Returns `{ updated, entryCount, categories: { categoryName: [{ title, observation, confidence, observations, lastSeen }] } }`
- Parsing approach: Read the markdown file. Extract frontmatter (updated, entry_count). Then parse the markdown structure:
  - H2 headings (`##`) are categories (e.g., "Organizational Patterns", "Thematic Patterns")
  - H3 headings (`###`) within a category are insight titles
  - Lines starting with `- **Observation:**` contain the observation text
  - Lines starting with `- **Confidence:**` contain the confidence float and metadata
  - Lines starting with `- **Observations:**` contain the observation count
  - Lines starting with `- **Last seen:**` contain the last-seen date
- If the file doesn't exist, return `{ updated: null, entryCount: 0, categories: {} }`.

**`writeInsights(vaultRoot, insightData)`** -- Writes the structured insight data back to `.claude/memory/insights.md`. Rewrites the entire file (not append -- same pattern as MEMORY.md). Format:
```markdown
---
updated: {today YYYY-MM-DD}
entry_count: {total entries across all categories}
---

# Long-term Insights

## {Category Name}

### {Insight Title}
- **Observation:** {observation text}
- **Confidence:** {0.0-1.0} (observed {N} times, last seen: {YYYY-MM-DD})
- **Observations:** {N}

...
```
Creates the directory and file if they don't exist.

**`distillInsights(vaultRoot, vaultIndex, tagIndex, options)`** -- The core insight distillation function. Claude calls this to analyze vault patterns and write findings.
- `options.activityLog` -- array of `{ action, path, date }` entries representing recent vault activity (passed by Claude based on session actions)
- Algorithm:
  1. Load current insights via `parseInsights(vaultRoot)`
  2. Analyze vault structure for organizational patterns:
     - Tag usage distribution: which tags have many notes vs few
     - Folder distribution: which folders are most active
     - Type distribution: which note types are most common
  3. Analyze tag-index for thematic patterns:
     - Tag clusters: groups of notes sharing 2+ tags
     - Growing topics: tags with many recent notes
  4. For each detected pattern:
     - Check if an existing insight matches (by observation text similarity -- simple substring/keyword match)
     - If existing: increment observation count, update confidence (+0.1, cap at 1.0), update lastSeen
     - If new: create with confidence 0.5, observations: 1, lastSeen: today
  5. Apply confidence decay: for each insight NOT reinforced in this pass, decrease confidence by 0.05 IF there was vault activity (per user decision: activity-relative decay, not calendar-based)
  6. Call `pruneInsights(insightData)` to remove stale entries
  7. Write back via `writeInsights(vaultRoot, insightData)`
  8. Return `{ added: count, reinforced: count, decayed: count, pruned: count, total: count }`
- Per user decision: threshold triggers use combined signals -- activity volume (10+ note changes) AND topic density (3+ notes in same tag cluster). Either signal meeting threshold triggers distillation.

**`pruneInsights(insightData)`** -- Removes low-quality insights.
- Remove entries with confidence < 0.3
- If total entries > 100 (soft limit per user decision), additionally remove the lowest-confidence entries until at or below 100
- Returns `{ pruned: count, remaining: count }`

**Project Memory:**

**`getActiveProjects(vaultRoot)`** -- Scans vault-index.json for project notes.
1. Load vault-index.json via `loadJson()`
2. Filter notes where: `note.frontmatter?.type === 'project'` AND path starts with `'01 - Projects/'` AND `note.isTemplate !== true`
3. Return `[{ path, name, status: note.frontmatter?.status || 'unknown' }]`

**`createProjectMemory(vaultRoot, projectNote)`** -- Creates a project-specific memory file.
- `projectNote`: `{ path, name, status }` from getActiveProjects
- Generates slug from project name: lowercase, replace spaces with `-`, remove non-alphanumeric except hyphens
- Creates `.claude/memory/project-{slug}.md` with structure:
```markdown
---
project: {Project Name}
source: {path to vault note}
status: {status from frontmatter or 'active'}
created: {today YYYY-MM-DD}
updated: {today YYYY-MM-DD}
---

# Project Memory: {Project Name}

## Current Status
{status} -- (context will be filled as Claude interacts with this project)

## Key Decisions
- (Decisions will be tracked here as they are made)

## Blockers
- (No current blockers)

## Next Actions
- (Next actions will be tracked here)

## Context
Source note: [[{Project Name}]]
```
- Does NOT overwrite existing file. Returns `{ created: true, path }` or `{ created: false, reason: 'already exists' }`.

**`archiveProjectMemory(vaultRoot, slug)`** -- Archives a project memory on completion and distills key lessons into insights.md. Per user decision: on project completion, archive AND distill key lessons.
1. Source: `.claude/memory/project-{slug}.md`
2. If source doesn't exist, return `{ archived: false, reason: 'file not found' }`
3. Read the source file content
4. **Distill lessons:** Extract key lessons from the project memory file:
   a. Parse the "Key Decisions" section -- each decision becomes a potential insight
   b. Parse the "Context" section -- look for patterns or takeaways
   c. For each extracted lesson, call `writeInsights` to add it to insights.md:
      - Category: "Project Lessons"
      - Title: `{Project Name}: {lesson summary}`
      - Confidence: 0.6 (higher than default 0.5 -- these are concrete observations, not heuristic patterns)
      - Observations: 1
      - LastSeen: today
   d. Load existing insights via `parseInsights(vaultRoot)` first, then merge new lessons and write back via `writeInsights(vaultRoot, insightData)`
5. Target: `.claude/memory/archive/project-{slug}.md`
6. Create archive directory if needed
7. Copy file to archive (not move -- per governance: never delete)
8. Update the archived file's frontmatter status to 'archived'
9. Return `{ archived: true, path, lessonsDistilled: count }` where count is the number of lessons written to insights.md

**Overview:**

**`generateMemoryOverview(vaultRoot)`** -- Generates the /memory dashboard output.
1. Read MEMORY.md (working memory summary)
2. Parse insights.md for category count and total entries
3. List active project memory files (glob `.claude/memory/project-*.md`)
4. Get embedding status (try to require embedder.cjs, handle if not yet available)
5. Format output:
```markdown
# Memory Dashboard

## Working Memory
{First 10 lines of MEMORY.md after frontmatter, or summary}

## Long-term Insights
- **Entries:** {N} across {M} categories
- **Categories:** {comma-separated list}
- **Top insights:** {2-3 highest-confidence insights, briefly}

## Project Memories
- {Project Name} ({status}) -- last updated {date}
- ...
(or "No project memories yet")

## Embedding Index
- **Notes embedded:** {N}
- **Last update:** {date}
(or "Not configured -- run `npm install` then `/scan` to enable semantic search")
```
Returns the formatted string.
