**Modify `.agents/skills/scan/scanner.cjs`:**

The scan() function is currently synchronous. Since syncEmbeddings is async, we need to add an async post-scan hook without changing the synchronous scan behavior for existing callers.

Add a new exported function:

**`async function scanWithEmbeddings(vaultRoot, options = {})`** -- Wraps scan() and adds embedding sync:
1. Call `const result = scan(vaultRoot, options)` (existing synchronous scan)
2. If `result.error`, return result immediately (scan failed, skip embedding)
3. Try to require `./../search/embedder.cjs` via `require('../search/embedder.cjs')`. Wrap in try/catch -- if the file doesn't exist yet (plan 01 not executed), skip embedding gracefully.
4. If embedder is available:
   a. Load vault-index.json via `loadJson(path.join(indexDir, 'vault-index.json'))`
   b. To get details for embedding sync: if `options.verbose` was NOT set, we need to pass the change classification to syncEmbeddings. The simplest approach: always pass scan result to syncEmbeddings. If `result.details` exists (verbose mode), use it directly. If not, the sync function handles it internally (falls back to hash comparison in the db).
   c. Call `const embedResult = await embedder.syncEmbeddings(vaultRoot, result, vaultIndex)`
   d. Attach embedding results to scan result: `result.embedding = embedResult`
5. Return the augmented result.

Important: The existing `scan()` export MUST remain unchanged and synchronous. `scanWithEmbeddings()` is an additional export. This ensures all existing callers (ensureFreshIndexes in other skills) continue to work without async changes.

Update `module.exports`:
```javascript
module.exports = { scan, scanWithEmbeddings };
```
