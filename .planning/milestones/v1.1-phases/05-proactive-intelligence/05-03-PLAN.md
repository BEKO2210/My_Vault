1. **`findTopicNotes(vaultRoot, topic)`** -- Async function. Discovers notes related to a topic using three signal layers:
   - **Tag matching:** Search tag-index.json for tags that contain the topic string (case-insensitive). Each matching note gets relevance 0.7.
   - **Title matching:** Search vault-index.json for notes whose name contains the topic (case-insensitive). Each match gets relevance 0.6. If a note already matched via tag, boost its relevance by 0.3 (capped at 1.0).
   - **Semantic search (optional):** Try to load `../search/embedder.cjs` and `../search/search-utils.cjs`. If embeddings are available (`isEmbeddingAvailable()`), generate a query embedding, get all embeddings from the DB, and run `semanticSearch()`. For each semantic result, add relevance equal to `similarity * 0.5` (or boost existing entry). Wrap the entire semantic block in try/catch -- if it fails, tag + title results are sufficient.
   - Exclude: templates, notes in `06 - Atlas/`, notes in `.claude/`.
   - Return: Array of `{ path, name, relevance, source }` sorted by relevance descending.
   - Use a Map to deduplicate by path, boosting relevance when multiple signals match.

2. **`createSynthesisNote(vaultRoot, topic, content, sourceNotes)`** -- Creates a new synthesis note file with proper frontmatter and structure.
   - `topic`: string -- the topic name (e.g., "Docker", "Productivity Systems").
   - `content`: string -- the synthesis body text generated by Claude (markdown format with `[[wiki-links]]`).
   - `sourceNotes`: array of `{ path, name }` -- the notes that were synthesized from.
   - Filename: `Zettel -- {topic}.md` placed in `03 - Resources/`.
   - Frontmatter (PROA-06 compliant):
     ```yaml
     ---
     type: zettel
     created: YYYY-MM-DD (today)
     updated: YYYY-MM-DD (today)
     tags:
       - synthesis
       - {topic-as-tag} (lowercased, spaces to hyphens, non-alphanumeric stripped)
     synthesized: true
     synthesized_by: claude
     source_notes:
       - "path/to/source1.md"

     status: active
     ---
     ```
   - Body structure:
     ```
     # Zettel -- {topic}

     > Claude-synthesized summary based on vault knowledge.
     ```

  <files>.agents/skills/synthesize/SKILL.md</files>
  <action>
Create `.agents/skills/synthesize/SKILL.md` following the established skill format:

**Frontmatter:**
```yaml
---
name: synthesize
trigger: /synthesize
description: Generate a topic summary from related vault notes with wiki-link citations and AI provenance metadata
version: 1.0.0
---
```

**Structure:**

1. **Title and description:** `# /synthesize -- Knowledge Synthesis` with summary explaining that Claude reads related notes and composes a new zettel that connects ideas with proper citations.

2. **Usage section:**
   ```
   /synthesize "Docker"                # Synthesize what the vault knows about Docker
   /synthesize productivity systems    # Multi-word topic synthesis
   /synthesize --top 5 "time mgmt"    # Limit to top 5 most relevant sources
   ```

3. **What It Does section** -- explain the three phases:
   - **Discover:** Find notes related to the topic via tags, titles, and semantic search (if available).
   - **Read:** Load full content of top-N related notes (default: top 10 by relevance).
   - **Synthesize:** Claude reads all source content and composes a coherent summary that connects ideas, identifies patterns, and cites sources via `[[wiki-links]]`. This is NOT a list of note summaries -- it is genuine synthesis that adds value.

4. **Execution Flow** -- numbered steps:
   1. Ensure fresh indexes: `ensureFreshIndexes('.')`
   2. Discover related notes: `findTopicNotes('.', topic)`
   3. If fewer than 2 notes found: "Not enough vault content about '{topic}' to synthesize. Found N note(s). Try a broader topic or add more notes first."
   4. Select top N notes (default 10, configurable via --top flag)
   5. Read full content of each selected note via `fs.readFileSync`
   6. Claude analyzes all source content and composes synthesis:
      - Extract key ideas from each note
      - Identify connections, patterns, and themes across notes
      - Write a coherent summary (not a bullet list of note summaries)
      - Include `[[wiki-links]]` to source notes as citations throughout the text
      - Every major claim must cite at least one source note
   7. Create synthesis note: `createSynthesisNote('.', topic, synthesisContent, sourceNotes)`
   8. Re-scan: `scan('.')` to update indexes with the new note
   9. Add synthesis note to relevant MOC if one exists (AUTO zone)
   10. Log creation to `.claude/changelog.md`

5. **Output Note Format** -- the generated note's frontmatter and structure:
   ```yaml
   ---
   type: zettel
   created: 2026-03-07
   updated: 2026-03-07
   tags:
     - synthesis
     - docker
   synthesized: true
   synthesized_by: claude
   source_notes:
     - "03 - Resources/Tool -- Docker.md"
     - "01 - Projects/Container Migration.md"
   status: active
   ---
   ```

6. **Quality Standards** -- what makes a good synthesis:
   - Every paragraph cites at least one source via `[[Note Name]]`
   - Synthesis connects ideas across notes (doesn't just summarize each note)
   - Total length: 200-500 words (concise but substantive)
   - Avoids repeating what any single note already says -- adds the "so what?" layer

7. **Governance section:**
   - READ-ONLY: Reading source notes, searching indexes
   - AUTO: Creating the new synthesis note, adding to MOC
   - NEVER: Modifying source notes (the synthesis references them, never edits them)

8. **Edge Cases:**
   - Topic matches 0-1 notes: Report insufficient content, do not create a note.
   - Topic matches 50+ notes: Use top 10 by relevance, note how many were found.
   - Synthesis note already exists for topic: Append date suffix to filename to avoid overwriting.
   - Semantic search unavailable: Proceed with tag + title matching only, note reduced coverage.

9. **Utility Module** section: Reference `synthesize-utils.cjs` with function signatures.
