  <action>
Create `.agents/skills/triage/` directory and `triage-utils.cjs` with the following exported functions:

1. **`getInboxNotes(vaultRoot, vaultIndex)`** -- Collects all notes in `00 - Inbox/` that need triage.
   - Exclude: notes in `00 - Inbox/Daily Notes/` subfolder, the `Inbox.md` file itself, templates.
   - For each qualifying note, read full file content via `fs.readFileSync`.
   - Return array of objects: `{ path, name, content, frontmatter, tags, headings, links, hasType, hasTags }`.
   - `hasType` = true if frontmatter.type exists and is non-empty.
   - `hasTags` = true if allTags array has length > 0.

2. **`applyFrontmatterFix(vaultRoot, notePath, updates)`** -- Modifies frontmatter fields on a note file (AUTO zone action).
   - `updates` is a plain object like `{ type: "tool", created: "2026-03-07", tags: ["docker", "devops"] }`.
   - If the note has no frontmatter block (`---` delimiters), insert a new frontmatter block at the top of the file.
   - If the note has existing frontmatter, update/add the specified fields without disturbing other fields or the note body.
   - Handle both scalar values (`type: tool`) and array values (`tags: [...]`) correctly in YAML output.
   - NEVER modify anything outside the frontmatter block -- the note body text must remain untouched.
   - Return `{ applied: true, fields: Object.keys(updates) }` on success.

3. **`moveNote(vaultRoot, sourcePath, targetFolder)`** -- Moves a note file from one folder to another (PROPOSE zone -- only call after user approval).
   - Construct target path as `targetFolder/basename(sourcePath)`.
   - Create target directory if it doesn't exist (`fs.mkdirSync` with `recursive: true`).
   - Check for filename collision at target -- if file exists, return `{ moved: false, reason: "File already exists at ..." }`.
   - Use `fs.renameSync` to move the file.
   - Return `{ moved: true, from: sourcePath, to: targetPath }`.

4. **`getTargetFolder(type)`** -- Returns the canonical target folder for a given note type using the same mapping as create-utils.cjs getTemplateInfo.
   - Use `require('../create/create-utils.cjs').getTemplateInfo(type)` to get the folder.
   - Return the folder string, or `null` if type is unknown.
   - This is a thin wrapper to keep the mapping consistent with /create.

**Implementation details:**
- `'use strict';` at top, `module.exports = { ... }` at bottom.
- Use `require('../scan/utils.cjs')` for `loadJson`.
- Use `require('../create/create-utils.cjs')` for `getTemplateInfo`.
- Use Node.js built-in `fs` and `path` only.
- Include JSDoc comments on all exported functions.
- The frontmatter regex pattern should handle both `\n` and `\r\n` line endings (Windows compatibility).

Create `.agents/skills/triage/SKILL.md` following the established skill format:

**Frontmatter:**
```yaml
---
name: triage
trigger: /triage
description: Classify inbox notes by type, auto-tag high-confidence items, and propose folder moves for review
version: 1.0.0
---
```

**Structure:**

1. **Title and description:** `# /triage -- Inbox Triage` with summary explaining the three-tier action model (AUTO/PROPOSE/review-only).

2. **Usage section:**
   ```
   /triage              # Triage all inbox notes
   /triage --dry-run    # Show classifications only, no changes applied
   ```

3. **Classification Approach section** -- explain that Claude reads each note's full content and uses its own reasoning to determine type. No NLP library or keyword matching. Classification signals include:
   - Headings structure (e.g., "## Installation" suggests tool)
   - Content patterns (e.g., decision pros/cons suggests decision)
   - Existing partial frontmatter
   - File naming patterns (e.g., "Book -- " prefix)
   - Links and tags already present

4. **Confidence Levels and Governance Mapping** -- the core table:
   | Confidence | Score | Governance Zone | Action |
   |------------|-------|-----------------|--------|
   | High | >= 0.8 | AUTO | Fill missing frontmatter (type, created, tags). Log to changelog. |
   | Medium | 0.5-0.8 | PROPOSE | Present classification and target folder. Wait for user approval. |
   | Low | < 0.5 | REVIEW | Show the note with Claude's best guess. No action taken. |

5. **Execution Flow** -- numbered steps:
   1. Ensure fresh indexes: `ensureFreshIndexes('.')`
   2. Load vault-index.json
   3. Get inbox notes: `getInboxNotes('.', vaultIndex)`
   4. If no inbox notes: report "Inbox is clear -- nothing to triage."
   5. For each inbox note, Claude reads the full content and classifies:
      - Determine note type (from the 12 valid types)
      - Assign confidence (high/medium/low based on signal strength)
      - Determine target folder via `getTargetFolder(type)`
      - Suggest tags based on content analysis
   6. Present triage results grouped by confidence:
      - HIGH: "Auto-applying: [[Note]] -> type: tool, tags: [docker, devops]"
      - MEDIUM: "Proposed: Move [[Note]] to 03 - Resources/ (classified as tool, medium confidence) -- approve?"
      - LOW: "Review: [[Note]] might be a zettel (low confidence) -- what type is this?"
   7. Apply AUTO actions: call `applyFrontmatterFix()` for high-confidence items
   8. Collect user decisions on PROPOSED items, execute approved moves via `moveNote()`
   9. Re-scan: call `scan('.')` after all modifications
   10. Log actions to `.claude/changelog.md`

6. **Governance section** -- explicit mapping:
   - AUTO: Fill missing frontmatter type, created date, tags (same as governance rule "fill missing frontmatter fields")
   - PROPOSE: Move note to target folder, rename note to match conventions
   - NEVER: Delete notes, edit note body text, merge notes

7. **Output Format** -- example triage report:
   ```
   ## Inbox Triage -- 5 notes

   **Auto-applied** (high confidence)
   - [[Docker Notes]] -- type: tool, tags: [docker, containers] -> filled frontmatter
   - [[Meeting with Sarah]] -- type: meeting, created: 2026-03-05 -> filled frontmatter

   **Proposed** (medium confidence)
   - Move [[Docker Notes]] from Inbox to 03 - Resources/? (tool)
   - Move [[Meeting with Sarah]] from Inbox to 01 - Projects/? (meeting)

   **Needs Review** (low confidence)
   - [[Random Thoughts]] -- could be zettel or daily? Please classify.

   Approve proposed moves? (yes/no/select)
   ```

8. **Edge Cases:**
   - Empty inbox: "Inbox is clear -- nothing to triage."
   - Note already has complete frontmatter: Skip classification, only check folder placement.
   - Note with `type: daily` in inbox root (not Daily Notes/): Propose move to Daily Notes subfolder.
   - Very short notes (< 50 chars body): Flag as low confidence regardless of signals.

9. **Utility Module** section: Reference `triage-utils.cjs` with function signatures.
