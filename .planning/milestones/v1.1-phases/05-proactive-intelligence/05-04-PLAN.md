Create `.agents/skills/maintain/` directory and `maintain-utils.cjs` with the following exported functions:

**Constants at module top:**
```javascript
const VALID_TYPES = ['project', 'area', 'resource', 'tool', 'person', 'meeting',
  'decision', 'code-snippet', 'zettel', 'daily', 'review'];
const VALID_STATUSES = ['active', 'planned', 'on-hold', 'completed', 'archived'];
const TYPE_FOLDER_MAP = {
  'project': '01 - Projects', 'area': '02 - Areas',
  'resource': '03 - Resources', 'tool': '03 - Resources',
  'zettel': '03 - Resources', 'person': '03 - Resources',
  'decision': '01 - Projects', 'meeting': '01 - Projects',
  'code-snippet': '03 - Resources',
  'daily': '00 - Inbox/Daily Notes', 'review': '00 - Inbox',
};
const DEFAULT_STALE_DAYS = 30;
```

**Exported functions:**

1. **`runConsistencyChecks(vaultIndex)`** -- Scans all notes for frontmatter and tag issues. Returns categorized issues object:
   ```javascript
   { critical: [...], warning: [...], info: [...] }
   ```
   Each issue: `{ type, path, name, fix, autoFixable, autoFix? }`.

   Checks to perform:
   - **Missing type** (critical, not autoFixable): Note has no `frontmatter.type`.
   - **Invalid type value** (warning, not autoFixable): `frontmatter.type` not in VALID_TYPES.
   - **Missing created date** (warning, autoFixable): No `frontmatter.created`. Auto-fix: derive from `note.mtime` as `YYYY-MM-DD`.
   - **Missing tags array** (warning, autoFixable): No `frontmatter.tags` or not an array. Auto-fix: `{ tags: [] }`.
   - **Type/folder mismatch** (info, not autoFixable): Note's folder doesn't match TYPE_FOLDER_MAP for its type. This is PROPOSE zone (structural move).
   - **Invalid status value** (warning, not autoFixable): `frontmatter.status` not in VALID_STATUSES.
   - **Tag casing** (warning, autoFixable): Tags that contain uppercase letters. Auto-fix: lowercase all tags.

   Exclude from checks: templates (`isTemplate`), `06 - Atlas/` notes, `.claude/` notes, system notes (Home, START HERE, Workflow Guide, Tag Conventions, Inbox, README).

2. **`getStaleProjects(vaultIndex, staleDays)`** -- Finds projects with `status=active` or `status=planned` and mtime older than `staleDays` (default: `DEFAULT_STALE_DAYS`).
   - Return array of `{ path, name, status, priority, lastModified (YYYY-MM-DD), daysSinceModified }` sorted oldest first.
   - Include suggested action: "Consider updating status to on-hold or archiving."

3. **`getOutdatedReferences(vaultIndex, linkMap)`** -- Detects notes that reference archived or completed items.
   - Scan link-map.json for resolved links where the target note has `status=completed` or `status=archived`.
   - Return array of `{ sourcePath, sourceName, targetPath, targetName, targetStatus, suggestion }`.
   - Suggestion: "[[Source]] links to [[Target]] which is {status}. Review if this reference is still relevant."
   - Also detect: notes in active projects that link to archived resources.

4. **`applyAutoFixes(vaultRoot, issues)`** -- Applies all `autoFixable: true` issues from runConsistencyChecks.
   - For each autoFixable issue, use a frontmatter modification approach (same logic as triage-utils.cjs applyFrontmatterFix -- reimplement here to avoid circular dependency since maintain may load before triage):
     - Read file content
     - Parse existing frontmatter block
     - Apply the `autoFix` field values
     - Write file back
   - NEVER modify anything outside the frontmatter block.
   - Return `{ fixed: number, details: [{ path, name, fieldsFixed }] }`.
   - Note: Reimplementing the frontmatter fix logic here (rather than importing from triage-utils) avoids a cross-skill dependency. The logic is small (~30 lines) and worth duplicating for independence.

**Implementation details:**
- `'use strict';` at top, `module.exports = { ... }` at bottom.
- Use `require('../scan/utils.cjs')` for `loadJson`.
- Use Node.js built-in `fs` and `path` only.
- Include JSDoc comments on all exported functions.
- Handle `\r\n` line endings in frontmatter parsing (Windows compatibility).
- The system note exclusion list should be a constant at module top for easy maintenance.

**Structure:**

1. **Title and description:** `# /maintain -- Vault Maintenance` with summary explaining the three-part audit (consistency, staleness, references) with governance-aware fixes.

2. **Usage section:**
   ```
   /maintain                  # Full maintenance report
   /maintain --fix            # Auto-apply all safe fixes (AUTO zone)
   /maintain --stale-days 60  # Custom staleness threshold (default: 30 days)
   ```

3. **What It Checks section** -- table of all check types:
   | Check | Severity | Auto-fixable | Governance |
   |-------|----------|--------------|------------|
   | Missing type | Critical | No (needs judgment) | PROPOSE |
   | Invalid type value | Warning | No (needs judgment) | PROPOSE |
   | Missing created date | Warning | Yes (from file mtime) | AUTO |
   | Missing tags array | Warning | Yes (add empty []) | AUTO |
   | Type/folder mismatch | Info | No (structural move) | PROPOSE |
   | Invalid status value | Warning | No (needs judgment) | PROPOSE |
   | Tag casing (uppercase) | Warning | Yes (lowercase) | AUTO |
   | Stale projects | Info | No (status decision) | PROPOSE |
   | Outdated references | Info | No (review needed) | READ-ONLY |

4. **Execution Flow** -- numbered steps:
   1. Ensure fresh indexes: `ensureFreshIndexes('.')`
   2. Load vault-index.json and link-map.json
   3. Run consistency checks: `runConsistencyChecks(vaultIndex)`
   4. Find stale projects: `getStaleProjects(vaultIndex, staleDays)`
   5. Find outdated references: `getOutdatedReferences(vaultIndex, linkMap)`
   6. Present structured report in priority order:
      - **Critical Issues** (missing type) -- needs user classification
      - **Auto-fixable Issues** (missing created, tag casing, missing tags)
      - **Proposed Fixes** (type mismatch, invalid values, folder moves)
      - **Stale Projects** -- with last-modified dates and suggested actions
      - **Outdated References** -- links to completed/archived notes
   7. If `--fix` flag: apply auto-fixes via `applyAutoFixes()`, report results
   8. Collect user decisions on proposed fixes
   9. Re-scan: `scan('.')` after any modifications
   10. Log actions to `.claude/changelog.md`

5. **Report Format** -- example output:
   ```
   ## Vault Maintenance Report

   **Critical** (1 issue)
   - [[Random Notes]] -- missing type field. What type is this note?

   **Auto-fixable** (3 issues)
   - [[Docker Notes]] -- missing created date (will set to 2026-02-15 from file date)
   - [[Meeting Notes]] -- tag "Docker" should be lowercase "docker"
   - [[Quick Idea]] -- missing tags array (will add empty [])
   Run /maintain --fix to apply these automatically.

   **Proposed** (2 issues)
   - [[Tool -- Git]] in 00 - Inbox/ should be in 03 - Resources/ (type: tool)
   - [[Old Project]] has status "wip" -- should be "active" or "planned"?

   **Stale Projects** (1 project)
   - [[Blog Platform]] -- active, last modified 2026-01-15 (51 days ago)
     Consider: update status to on-hold or archive?

   **Outdated References** (1 reference)
   - [[Current Workflow]] links to [[Old API]] (status: archived)
     Review if this reference is still relevant.

   Summary: 1 critical, 3 auto-fixable, 2 proposed, 1 stale, 1 outdated
   ```

6. **Governance section** -- explicit mapping:
   - AUTO: Fix missing created date (from mtime), lowercase tags, add empty tags array
   - PROPOSE: Type reclassification, status correction, folder moves, stale project status changes
   - NEVER: Delete files, edit body text, force-override user classifications
   - READ-ONLY: Outdated reference detection (informational only)

7. **Configuration section:**
   - Default stale threshold: 30 days (override with `--stale-days N`)
   - System notes excluded from checks: Home, START HERE, Workflow Guide, Tag Conventions, Inbox

8. **Edge Cases:**
   - Clean vault (no issues): "Vault is in good shape -- no issues found."
   - Many issues (50+): Show top 10 per category with "N more" summary.
   - Recently archived project still linked: Flag as outdated reference but note it may be intentional.

9. **Utility Module** section: Reference `maintain-utils.cjs` with function signatures.
